# 健身打卡 App — 项目说明书

## 项目目标

做一个让自己坚持健身的打卡工具：
- 每天一键打卡，记录"今天练了"
- 记录每次训练的动作、组数、重量
- 看到连续打卡天数，给自己成就感
- 第一步：网页版 → 第二步：手机可安装（PWA）→ 第三步：正式产品

## 用户背景

**开发者是编程零基础新手。**
- 代码注释要多，解释每一段在做什么
- 变量名要用中文拼音或清晰的英文单词，不要用 a/b/x 这种
- 每个文件的功能要一目了然
- 遇到新概念，在注释里用大白话解释

## 技术栈

| 工具 | 版本 | 用途 |
|------|------|------|
| React | 18+ | 搭积木式写页面，把页面拆成小组件 |
| Vite | 5+ | 开发服务器，保存代码立刻刷新页面 |
| Tailwind CSS | 3+ | 直接在 HTML 上写 class 来控制样式 |
| localStorage | 浏览器内置 | 存数据，不需要服务器，关了再开数据还在 |

## 第一阶段 MVP 功能清单

- [ ] 首页：显示今天日期 + 一键打卡按钮
- [ ] 打卡记录：过去 30 天日历，绿色=打卡，灰色=未打卡
- [ ] 训练记录：输入今天做的动作（名称 + 组数 + 重量）
- [ ] 数据本地存储：用浏览器 localStorage 保存，无需账号
- [ ] 连续打卡天数显示（"你已经连续打卡 X 天！"）

## 代码规范

```
// 好的注释示例：解释"为什么"和"是什么"
// 把今天的日期格式化成 "2026年5月24日" 这样的中文格式
const todayText = formatDate(new Date())

// 不好的注释（废话）：
// 格式化日期
const todayText = formatDate(new Date())
```

- 每个组件文件顶部写一行注释说明这个文件是干什么的
- 函数名用动词开头：`handleClick`（处理点击）、`saveRecord`（保存记录）
- 状态变量名要清晰：`isCheckedIn`（今天打卡了吗）、`workoutList`（训练列表）
